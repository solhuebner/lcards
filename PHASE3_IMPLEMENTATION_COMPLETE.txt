✅ TRANSFORMATION TYPES (UI FORMS)
═══════════════════════════════════

 1. unit_conversion     - Temperature, distance, speed conversions
 2. scale               - Map input ranges to output ranges
 3. exponential_smoothing - Noise reduction with alpha
 4. moving_average      - Average over N points
 5. clamp               - Min/max bounds
 6. offset              - Add/subtract constant
 7. multiply            - Scale by factor
 8. round               - Decimal precision
 9. delta               - Rate of change
10. expression          - Custom JavaScript

✅ AGGREGATION TYPES (UI FORMS)
════════════════════════════════

1. avg    - Moving average over window
2. min    - Minimum value over window
3. max    - Maximum value over window
4. sum    - Total over window
5. rate   - Rate of change per time unit
6. trend  - Rising/falling/stable detection

📋 EXAMPLE CONFIGURATION
════════════════════════

data_sources:
  temperature:
    entity: sensor.outdoor_temp
    transformations:
      fahrenheit:
        type: unit_conversion
        from_unit: celsius
        to_unit: fahrenheit
      smoothed:
        type: exponential_smoothing
        alpha: 0.3
      rounded:
        type: round
        decimals: 1
    aggregations:
      avg_5min:
        type: avg
        window_seconds: 300
      trend:
        type: trend
        window_seconds: 600
        threshold: 0.5

🔄 USER WORKFLOW
═══════════════

1. Open datasource dialog (add/edit mode)
2. Scroll to Transformations section
3. Click "Add Transformation"
4. Select type and fill form
5. Click "Create"
6. Repeat for aggregations
7. Click "Save" on datasource dialog
8. Configuration is persisted to YAML
