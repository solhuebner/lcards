HA CARD PICKER FIX - IMPLEMENTATION SUMMARY
================================================================================

PROBLEM:
--------
Home Assistant lazy-loads hui-card-picker and hui-card-element-editor.
On fresh page load, these components are undefined, causing MSD Studio
to fall back to a legacy dropdown picker. Even when components later
load (after opening HA's Grid editor), they render incorrectly due to
missing initialization properties.

SOLUTION:
---------
Implement eager loading mechanism with robust Lovelace access and
comprehensive error handling.

KEY CHANGES:
------------

1. NEW PROPERTY: _haComponentsAvailable (Boolean)
   - Tracks component availability state
   - Updated asynchronously on dialog initialization
   - Used to decide native vs legacy picker

2. NEW METHOD: _ensureHAComponentsLoaded() (Async)
   Location: After _getRealLovelace() method (~70 lines)
   
   Logic:
   a) Check if components already loaded → return true
   b) Create temporary hidden container
   c) Create temporary hui-card-picker element
   d) Wait 100ms for HA's lazy loading to trigger
   e) Wait up to 2 seconds for component definition
   f) Clean up temporary container
   g) Final check and return boolean result
   
   Called from: connectedCallback() (non-blocking async)

3. NEW METHOD: _getLovelace() (Enhanced)
   Location: After _ensureHAComponentsLoaded() (~40 lines)
   
   Fallback Chain:
   Path 1: this.hass?.connection?.lovelace ← Most Reliable
   Path 2: ha-panel-lovelace.lovelace (Shadow DOM navigation)
   Path 3: home-assistant.lovelace (Direct property)
   Path 4: window.lovelace (Deprecated but supported)
   Path 5: this._getRealLovelace() (Existing implementation)
   
   Returns: Lovelace instance or null

4. NEW METHOD: _debugHAComponents() (Diagnostics)
   Location: After _getLovelace() (~30 lines)
   
   Logs:
   - Component availability flags
   - Component registration status
   - Lovelace instance availability
   - Configuration presence
   - HASS state count
   
   Error Reporting:
   - Lists all attempted Lovelace access paths on failure

5. UPDATED: connectedCallback()
   Added async call to _ensureHAComponentsLoaded()
   Updates _haComponentsAvailable on success/failure
   Non-blocking - doesn't delay dialog opening

6. UPDATED: _renderControlFormCard()
   - Calls _debugHAComponents() for diagnostics
   - Uses _haComponentsAvailable flag instead of direct check
   - More reliable component detection

7. UPDATED: _renderControlFormCardNative()
   - Uses _getLovelace() for robust access
   - Adds .cardConfig={{}} to hui-card-picker
   - Adds .disabled=${false} to hui-card-element-editor
   - Enhanced logging with lovelace availability
